# Super Powers

## Operating Rule

Use available skills and workflows deliberately. If a relevant skill exists, read it before acting and follow it unless the user's explicit instructions conflict.

## Workflow

1. Check for applicable skills before doing task work.
2. State the skill or workflow being used and why.
3. For multi-step work, write or maintain a short checklist.
4. Keep the task bounded to the user's request.
5. Verify with fresh evidence before claiming success.
6. Report what changed, what was verified, and any remaining risk.

## Skill Priority

1. Use process skills first when they define the approach: brainstorming, planning, debugging, review, verification.
2. Use domain or implementation skills second: frontend, documents, APIs, tests, deployment, or project-specific workflows.
3. If skills conflict, follow the user's explicit instructions first, then the more specific skill, then the broader process skill.

## Completion Discipline

Before saying work is done:

- Re-read the request.
- Check the relevant files or artifacts.
- Run the most direct verification command available.
- Read the output and report the actual result.

Never substitute confidence for evidence.
